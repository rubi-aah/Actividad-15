# Guía de Despliegue

Este documento explica cómo poner en funcionamiento el sistema web utilizando GitHub Pages.

---

## Requisitos

- Tener una cuenta en GitHub
- Tener un repositorio creado
- Tener los archivos del proyecto en la carpeta /docs

---

## Pasos para el despliegue

1. Crear un repositorio en GitHub.

2. Subir los archivos del proyecto al repositorio.

3. Crear una carpeta llamada /docs dentro del repositorio.

4. Agregar los archivos .md dentro de la carpeta /docs.

5. Ir a la configuración del repositorio (Settings).

6. Buscar la sección "Pages".

7. En "Branch", seleccionar la rama "main" y la carpeta "/docs".

8. Guardar los cambios.

9. Esperar unos minutos a que GitHub genere el sitio web.

10. Acceder al sitio web mediante la URL proporcionada por GitHub.

---

## Actualización del sistema

Cada vez que se realiza un cambio en los archivos y se hace un commit, el sitio se actualiza automáticamente en pocos minutos.

Esto permite mantener la documentación siempre actualizada sin necesidad de procesos manuales.
